title: Shader

在[着色器简介](../intro)我们了解了着色器的基本概念，Galacean 引擎在着色器程序的基础上，将其他渲染相关的信息一起进行了统一的封装，即 Shader 对象。Shader 对象和 [材质](../../material/material)一起决定了被渲染物体最终的渲染结果。


## Shader

Shader 对象包含以下内容:

- name

Shader 实例对象名。Shader 对象创建后会被引擎缓存，后续复用可以通过 `Shader.find()` 以 `Shader.name` 为索引进行查找，因此引擎规定 Shader 的 `name` 属性在单个引擎实例下必须唯一。

- SubShader

一个 Shader 对象包含至少一个 SubShader 对象。

## SubShader

一个着色器程序可能会运行在不同的GPU硬件平台、不同的渲染管线中，因此 Shader 对象通过 tags 键值对的方式为不同的硬件平台、渲染管线指定不同的渲染逻辑，即 SubShader 对象。

SubShader 包含以下内容:

- Tags

指定该 SubShader 兼容的运行环境，如渲染管线、硬件平台等

- Passes

一个 SubShader 包含至少一个 ShaderPass 对象。

- RenderStates

SubShader 下所有 ShaderPass 共享的 [RenderState 渲染状态](../../material/composition/#渲染状态)

## ShaderPass

ShaderPass 封装了具体的着色器程序，以及执行最终渲染时的[渲染状态](../../material/composition/#渲染状态)。

<Callout type="info">
同一 SubShader 下的 ShaderPass 按照数组顺序依次渲染，渲染效果依次叠加形成 Shader 在当前帧的最终渲染结果
</Callout>
